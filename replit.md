# NHMRC Archive Redirect (Drupal 10 module)

A custom Drupal 10 module that redirects requests for unpublished ("archived") nodes to configured landing pages per content type, with optional stored redirects on node delete.

## Run & Operate

- Workflow `Start application` runs PHP's built-in server on port 5000 against the Drupal site at `drupal/web/`.
- Command: `PHP_CLI_SERVER_WORKERS=4 php -S 0.0.0.0:5000 -t /home/runner/workspace/drupal/web /home/runner/workspace/drupal/web/router.php`
- Drush: `cd drupal && ./vendor/bin/drush <cmd>` (e.g. `drush cr`, `drush en nhmrc_archive_redirect -y`).
- Default admin login: user `admin` / pass `admin`. Module config UI: `/admin/config/content/archive-redirects`.

## Stack

- PHP 8.2 (Nix module `php-8.2`).
- Drupal 10 (`drupal/recommended-project`) with Drush 13, installed via Composer in `drupal/`.
- SQLite database at `drupal/data/drupal.sqlite`.
- The custom module lives in this repo at `nhmrc_archive_redirect/` and is symlinked into `drupal/web/modules/custom/nhmrc_archive_redirect`.

## Where things live

- `nhmrc_archive_redirect/` — the actual module source (this is the repo's product).
  - `nhmrc_archive_redirect.module` — hooks (cache invalidation, predelete/delete).
  - `src/EventSubscriber/` — runtime redirect subscribers.
  - `src/Form/ArchiveRedirectSettingsForm.php` — admin settings form.
  - `config/install/`, `config/schema/` — default config and schema.
  - `README.md` — full module documentation (source of truth for behavior).
- `drupal/` — local Drupal site used to host/run the module (not part of the deliverable module).
- `drupal/web/router.php` — small router so PHP's built-in server serves Drupal correctly.
- `drupal/web/sites/default/settings.php` — appended Replit settings: `trusted_host_patterns = ['.*']`, reverse proxy enabled.

## Architecture decisions

- The repo only contains a Drupal module, not a full site. A scaffolded Drupal 10 site under `drupal/` is used purely as a **development host** so the module can be exercised in Replit. This is not a production-hardened Drupal site.
- `trusted_host_patterns = ['.*']` and PHP's built-in server are used because this is a dev/demo environment proxied by Replit. For real production hosting, replace with nginx/apache + php-fpm and pin host patterns to actual domains.
- SQLite chosen instead of MySQL/Postgres for zero-config local development.
- The module is symlinked (not copied) into the host site so edits in `nhmrc_archive_redirect/` are picked up immediately (after `drush cr`).

## Product

- Per-bundle configurable redirects (301/302) for unpublished nodes, with a fallback path.
- Optional stored redirects on node deletion (path-prefix rules).
- Cache integration with Internal Page Cache and Dynamic Page Cache via `LocalRedirectResponse` and node cache tags.

## Gotchas

- `drupal/` is generated by Composer; don't edit module code there — edit in `nhmrc_archive_redirect/` (symlinked).
- After changing module code, run `cd drupal && ./vendor/bin/drush cr` to clear caches.
- The PHP built-in server is single-threaded by default; `PHP_CLI_SERVER_WORKERS=4` enables concurrent requests.
